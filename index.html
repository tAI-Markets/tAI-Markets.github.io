Key Features

Built for the AI Economy

Every component designed to bridge decentralized finance with AI compute.

🔄

Standardization

AI inference from any provider becomes fungible ERC-20 tokens. A tGLM5 token is a tGLM5 token, regardless of who provided the compute.

🔒

Collateralization

150% minimum collateral ratio in USDC. Providers must lock collateral to mint tokens, ensuring redemption guarantees.

🎯

Smart Routing

Waterfall algorithm routes redemption requests based on Service Factor (σ) — quality, latency, and cost optimization.

⚡

Automated Liquidation

Keepers monitor collateral ratios and trigger liquidations when CR falls below 150%, protecting token holders.

🛟

Provider of Last Resort

When all providers fail, the PLR module sources liquidity via x402 protocol, ensuring 100% settlement guarantee.

🏛️

DAO Governance

tModel holders govern protocol parameters, fee structures, and whitelisted providers through on-chain voting.

How It Works

From Collateral to Settlement

A complete cycle of AI inference tokenization and redemption.

1

Provider Locks Collateral

AI inference providers deposit USDC into CDPVaults at 150% collateral ratio. This collateral backs the tokens they mint, ensuring redemption guarantees for token holders.

2

Mint tModel Tokens

Providers mint tModel tokens (e.g., tGLM5, tOpus46) representing 1 million inference tokens. These ERC-20 tokens are now tradeable on DEXs or used for redemption.

3

User Redeems for Inference

Token holders burn tModel tokens to request AI inference. The RoutingEngine assigns the request to the best provider based on Service Factor (σ), optimizing for quality and cost.

4

Settlement & Fees

The provider delivers inference, settlement occurs with a 5% fee to the DAO treasury. If providers fail, the PLR module ensures completion via x402 protocol integration.
